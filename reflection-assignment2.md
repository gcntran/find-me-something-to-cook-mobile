I faced several challenges throughout this assignment, especially because I decided to build a more dynamic and feature‑rich mobile app instead of following the simpler examples. I think this assignment is the most difficult one I ever did. My original plan was to recreate the core experience of my old version, which meant working with API data, along with creating more screens, adding new modal feature, updating navigation. This made the project more complex, but it also pushed me to learn a lot more than I expected.

One of the biggest challenges was understanding how Expo Router works. I had to convert and transfer my old components and screens from previous assignment to this assignment because I wat to keep the same concept and see how I can improve the app. I had to figure out how to link the tabs and stack with the old folder structure, and how to use dynamic routes, modal routes, and tab navigation all at the same time. I also struggled with restoring headers after hiding them globally, and I had to manually enable them for specific screens like the recipe details and settings sections. Managing navigation between screens, especially when passing parameters like recipe IDs, took a lot of trial and error. I used global URL parameters for this assignment which was not taught on BrightSpace (or I didn't pay attention to the lectures). I also didn't add information that will be fetched from the API for the recipe detail because I don't have enough time.

Another challenge was building the Notebook feature. Saving recipes, deleting them, and keeping everything synced across screens required using React context. I had to debug issues with IDs not matching, state not updating, and the delete modal not working (it's still not working, I gave up). These problems were frustrating, but solving (some of) them helped me understand state management much better. I also learned how to add a simple toggle as a switch for the settings screens instead of using the old dark mode toggle since that is more complex to edit.

I (again) relied on AI support throughout the assignment, especially when I got stuck on navigation, context, or styling issues. It helped me understand concepts (mostly everything since I complicated myself with the app) I wasn’t familiar with and guided me through debugging.

Overall, this assignment was challenging but rewarding. I pushed myself to build something more complex than the examples, and even though it took more time and effort, I learned a lot about React Native, Expo Router, and mobile UI design. For the future assignment, I want to continue improving the UX and polish the theme system for any app to feels even more consistent and professional.
